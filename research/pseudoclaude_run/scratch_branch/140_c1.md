This search endpoint is slow and I think it's a query-plan issue. Can you just optimize it? Don't change the interface.

```python
def search_users(name_filter, limit=50):
    conn = get_db_connection()
    cursor = conn.cursor()
    query = "SELECT id, name, email FROM users WHERE name LIKE '%{}%' ORDER BY created_at DESC LIMIT {}".format(name_filter, limit)
    cursor.execute(query)
    return cursor.fetchall()
```
